Architecture Diagram

Command Flow — /sticker

1. User sends image + "/sticker"
2. Baileys captures image buffer + command text
3. Baileys emits event → NestJS: { from, imageBuffer, command }
4. MessageGateway receives → passes to CommandParser
5. CommandParser routes to sticker handler
6. Sticker handler calls PythonClientService → POST /sticker
7. Python: resize 512×512, convert to WebP, return bytes
8. ReplyService calls BaileysService.sendMessage(sticker)
9. User receives the sticker ✓

Key Notes

Baileys ↔ NestJS — use Node.js EventEmitter (same process, no HTTP needed)
NestJS ↔ Python — use multipart/form-data to pass image buffers over HTTP
Baileys session — store auth state to disk or Redis so bot survives restarts
rembg first run — downloads AI model (~170MB), pre-download on deploy
Sticker format — WhatsApp requires WebP at exactly 512×512 and under 500KB
